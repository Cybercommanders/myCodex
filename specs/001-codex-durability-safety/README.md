## The work in one paragraph
`state.json` is mutated by multiple OS processes through an unsynchronized
read-modify-write with non-atomic writes and a silent reset on corrupt reads. The fix
is a per-workspace **atomic-`mkdir` directory lock** (re-read inside the lock,
TTL+host+PID reclaim that's TOCTOU-safe and SIGKILL-safe), **atomic+durable writes**
(O_EXCL temp → fsync file → rename → fsync dir), and **non-destructive,
locked-only corrupt quarantine**. Plus four smaller fixes: a precise process matcher
with a uid guard (no more killing root `earlyoom`), EPERM-as-not-delivered in both
kill branches, a Stop-gate that fails open only for genuinely-ran-but-odd reviews
(never a crash) and always prints a bypass, and a `/codex:init` effort-knob
reconciliation (`max` → `xhigh`, actually forwarded). Filesystem-only, no new deps, no
schema change, no migration.
